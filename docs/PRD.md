# Product Requirements Document (PRD)
## VK-Book Android - Мобильная экосистема цифровой трансформации

**Версия:** 1.0  
**Дата:** Февраль 2026  
**Автор:** Technical Product Manager

---

## 1. Проблема (Problem Statement)

### Текущее состояние

Операционная деятельность на реакторной установке ВК-50 сталкивается с критическими проблемами:

#### 1.1 Информационный хаос
- Сменный персонал тратит **десятки минут** на поиск данных в бумажных журналах
- Разрозненные Excel-таблицы хранятся в разных местах
- Отсутствие единого источника истины для технических данных
- Высокая вероятность ошибок при ручном поиске информации

#### 1.2 Операционные риски
- Огромное количество оборудования (тысячи единиц арматуры)
- Ошибки навигации у новичков из-за сложности объекта
- Ошибки по невнимательности у опытных сотрудников
- Пропуски замеров из-за человеческого фактора

#### 1.3 Разрыв коммуникаций
- Отсутствие мобильного доступа к схемам в зонах без стационарных ПК
- Необходимость возвращаться к рабочему месту для получения информации
- Замедление технологического процесса из-за отсутствия данных на месте

## 2. Целевая аудитория

### 2.1 Основные пользователи

1. **Операторы реакторной установки**
   - Ежедневная работа с оборудованием
   - Необходимость быстрого доступа к техническим данным
   - Работа в экранированных зонах без связи

2. **Обходчики оборудования**
   - Регулярные проверки арматуры
   - Работа с техническими схемами
   - Необходимость мобильного доступа к данным

3. **Инженерный персонал**
   - Контроль графиков проверок
   - Анализ данных оборудования
   - Управление расписанием смен

4. **Администраторы системы**
   - Редактирование данных арматуры
   - Загрузка обновлений на сервер
   - Управление конфигурацией системы

## 3. Цели продукта

### 3.1 Бизнес-цели

- **Сокращение времени доступа к данным** с десятков минут до менее 60 секунд
- **Исключение ошибок навигации** через интуитивный интерфейс и поиск
- **Мобильный доступ к схемам** в зонах без стационарных ПК
- **Автоматизация процессов** проверки оборудования

### 3.2 Технические цели

- **Offline-first архитектура** для работы в экранированных зонах
- **Высокая производительность** при работе с большими объемами данных
- **Надежность** через автоматическое тестирование
- **Масштабируемость** для расширения функционала

## 4. Функциональные требования

### 4.1 Модуль "Арматура"

**Описание:** База данных арматуры с поиском и просмотром схем

**Требования:**
- Поиск арматуры по номеру, названию, характеристикам
- Отображение координат на интерактивных схемах
- Просмотр детальной информации об арматуре
- Экспорт/импорт данных в Excel формате
- Постраничная загрузка для оптимизации производительности

**Критерии приемки:**
- Поиск арматуры выполняется менее чем за 5 минут
- Данные загружаются из локального кэша при отсутствии связи
- Поддержка работы с файлами размером до 50 МБ

### 4.2 Модуль "Сигналы БЩУ"

**Описание:** Справочник сигналов блочного щита управления

**Требования:**
- Полнотекстовый поиск по номеру и описанию сигнала
- Быстрый доступ к техническим параметрам
- Фильтрация по типам сигналов

**Критерии приемки:**
- Время доступа к данным менее 60 секунд
- Работа в офлайн-режиме

### 4.3 Модуль "Данные"

**Описание:** Просмотр больших Excel таблиц с кэшированием

**Требования:**
- Просмотр Excel файлов размером до 50 МБ
- Кэширование для быстрой работы
- Постраничная загрузка данных
- Поддержка форматов .xlsx и .xls

**Критерии приемки:**
- Открытие файла менее чем за 10 секунд
- Плавная прокрутка больших таблиц
- Экономия памяти через постраничную загрузку

### 4.4 Модуль "Расписание"

**Описание:** Календарь смен на годы вперед (2025-2030)

**Требования:**
- Визуализация графика смен на год
- Автоматический расчет графика в обе стороны
- Заметки к дням для дополнительной информации
- Экспорт/импорт расписания в JSON формате

**Критерии приемки:**
- Расчет графика выполняется менее чем за 1 секунду
- Поддержка планирования до 2030 года
- Сохранение расписания локально

### 4.5 Модуль "График проверок"

**Описание:** Расписание проверок оборудования по часам с подсветкой текущего часа

**Требования:**
- 36-часовая шкала времени (20:00 предыдущего дня - 07:00 следующего)
- Система напоминаний с гибкими правилами повторения
- Личные и служебные задачи с разделением
- Подсветка активных задач в реальном времени
- Загрузка из Excel/CSV с автоматическим парсингом

**Критерии приемки:**
- Подсветка текущего часа работает в реальном времени
- Напоминания срабатывают точно в указанное время
- Исключение пропусков замеров через визуальную индикацию

### 4.6 Модуль "Настройки"

**Описание:** Управление конфигурацией системы

**Требования:**
- 6 профессиональных тем оформления
- Настройка подключения к серверу
- Управление автосинхронизацией
- Выбор видимых вкладок
- Управление административным доступом

**Критерии приемки:**
- Применение темы без перезапуска приложения
- Сохранение настроек локально
- Безопасное хранение учетных данных администратора

## 6. Ограничения и зависимости

### 6.1 Технические ограничения

- Минимальная версия Android: 8.0 (API 26)
- Требуется доступ к файловой системе для кэширования
- Необходимость работы в офлайн-режиме

### 6.2 Бизнес-ограничения

- Защита интеллектуальной собственности
- Соответствие требованиям безопасности предприятия
- Ограничения на использование внешних библиотек

### 6.3 Зависимости

- Серверное API для синхронизации данных
- Excel файлы с техническими данными
- PDF схемы оборудования
